# 双线测试模式及大规模扩题 Spec

## Why
目前的测试混合了情感、职场、家庭场景。为了让用户体验更聚焦，同时满足部分只关注“鉴别渣男/渣女”用户的需求，需要将网站拆分为两种独立的测试模式。并且，原有题量（30题）偏少，需要大幅扩充，使测试结果更精准、更具权威性，同时要求图鉴文案更加犀利、抽象、有梗（如“吸渣体质”），且界面必须完美适配手机端到 1920px 宽屏 PC 端。

## What Changes
- **新增双测试入口**：首页改造为双模式入口。
  - **模式一：赛博恋爱段位测试（原防渣测试，名字温和化）**（默认主推）。总题量 40 题（男女目标各 20 题）。题目需涵盖从“暧昧/朋友”、“交往中”、“订婚/未婚妻(夫)”、“已婚”、“前任”等不同情感阶段的典型 PUA 场景。
  - **模式二：全景防PUA综合测试**。总题量 50 题。合理分配比例（如：15职场、15家庭、10社交、10情感），全面考察社会防御力。
- **BREAKING**: 更新 `types/index.ts` 中的 `StoreState`，加入 `testMode: 'romance' | 'full' | null` 字段。更新 `Question` 接口，加入可选的 `relationshipStage` 字段以支持恋爱阶段划分。
- **图鉴文案极致抽象化**：在 `scoring.ts` 中，为专门的情感模式和全景模式准备极具攻击性和网感的图鉴点评（如“绝世大冤种”、“纯爱战神”、“吸渣体质”、“顶级海王”等）。
- **全端响应式优化**：重构组件的 Tailwind CSS 容器类，使用 `max-w-screen-xl`、`xl:grid-cols-2` 等类名，确保在 1920px 分辨率下的居中和高清显示，同时保持移动端的沉浸式体验。

## Impact
- Affected specs: `src/types/index.ts`, `src/store/useStore.ts`, `src/utils/scoring.ts`.
- Affected code: `src/pages/Home.tsx` (入口改造), `src/pages/Assessment.tsx` (题目筛选逻辑), `src/components/ChatSimulation.tsx` (响应式), `src/data/questions.ts` (海量扩题).

## ADDED Requirements
### Requirement: 模式分离与题目筛选
系统应当允许用户在首页选择测试模式，并根据选择的模式（及性别，若为恋爱模式）动态加载对应的题库。

#### Scenario: 选择赛博恋爱段位测试
- **WHEN** 用户点击“赛博恋爱段位测试”，并选择“我是女生”
- **THEN** 系统加载 20 道专为女性设计的恋爱场景题，涵盖朋友、男友、未婚夫、老公、前任等各个阶段的话术。

### Requirement: 超大屏 (1920px) 响应式适配
系统应当在大屏下依然保持极佳的阅读体验和新粗野主义（Neo-Brutalism）的排版张力。

#### Scenario: 1920px 显示器下查看图鉴
- **WHEN** 用户在 1920x1080 的 PC 端打开 `/gallery` 页面
- **THEN** 网格布局应当自动展开为 4-5 列，卡片居中对齐且不会被无限拉伸，边缘留白合理，整体视觉极具冲击力。
