# 反PUA网站（SBTI测试） Spec

## Why
当代年轻人在恋爱、交友、职场中常常遇到情感操控（PUA）。很多人在不知不觉中成为了被操控的对象（俗称“大冤种”）。
通过一个具有讽刺意味、画风夸张且极具互动性的“反PUA”测试网站（类似于MBTI但更抽象、更接地气的“SBTI”测试），能让用户在做题和模拟聊天的娱乐过程中，认清各种PUA话术和自己的性格弱点，从而提高防备心。

## What Changes
- 初始化 React + Vite + Tailwind CSS 项目。
- 设计并实现 Neo-Brutalism（新粗野主义）风格的 UI 框架，使用粗黑边框、高对比度色彩和硬阴影。
- 实现首页，包含夸张的标语和开始测试按钮。
- 开发测试引擎，支持两种题型：
  - 常规选择题：测试用户在特定情境下的反应。
  - 模拟聊天小游戏：模拟真实的微信聊天场景，让用户选择如何回复对方的“套路”或“打压”。
- 实现四维评分系统（如 S-B-T-I 轴），根据用户的选择计算分数。
- 开发结果展示页，动态展示生成的四字“防PUA人格”，并提供具体的解读和防骗建议。

## Impact
- Affected specs: 全新的独立前端项目。
- Affected code: 初始化所有基础架构、组件、页面路由和题库数据文件。

## ADDED Requirements
### Requirement: 测试引擎与评分
系统应当能够渲染包含多种题型的测试问卷，并在用户每次选择后，累加四个独立维度的分数。

#### Scenario: 模拟聊天答题
- **WHEN** 用户进入一道“模拟聊天”题
- **THEN** 界面应显示类似手机聊天气泡的对话历史，并在底部提供几个回复选项供用户点击。点击后触发发送动画并进入下一题，同时后台记录该选项对应的维度得分。

### Requirement: 结果计算与展示
系统应当根据用户的最终得分，计算并展示四字母结果。

#### Scenario: 结果生成
- **WHEN** 用户完成最后一道题
- **THEN** 系统根据每个维度的正负分，计算出对应的四个字母（如：S、B、T、I），并跳转到结果页，展示炫酷的大字号结果以及对应的人格解析。
